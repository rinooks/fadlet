PROBLEM

워크숍 운영, 이런 고민 한 번쯤 해보셨나요?

패들렛, 미로, 뮤랄을 써봤지만 무언가 부족하다고 느끼셨다면 — Fadlet이 그 자리를 채웁니다.

PAIN 01

"가입하라고 하면
참여자가 이탈해요"

워크숍 시작 전 5-10분이 매번 가입 안내로 사라집니다. 30대 이상 학습자는 그 자리에서 포기하기도 합니다.

PAIN 02

"카카오 오픈채팅방
매번 만드느라 분주해요"

차수마다 새 오픈채팅방 생성, 링크 공유, 입장 안내, 끝나면 정리까지. 본 워크숍보다 채팅방 관리에 시간을 더 씁니다.

PAIN 03

"보드 보랴, 채팅 보랴
정신이 없어요"

줌으로 진행하면서 보드는 패들렛, 채팅은 카톡, 자료는 메일로. 운영자 한 명이 4개 도구를 동시에 관리합니다.

PAIN 04

"한국어가 어색해서
설명을 또 해야 해요"

"칠판 추가" "섹션 만들기" 같은 번역체 메뉴가 학습자를 헷갈리게 합니다. 결제도 해외 카드로만, 세금계산서도 어렵습니다.

SOLUTION

하나의 화면에서,
워크숍의 모든 흐름을.

참여자의 보드 작성, 학습자 간 실시간 소통, 운영자의 진행 관리 — 분산되어 있던 워크숍의 세 흐름을 하나로 통합했습니다.

KEY FEATURES

운영자가 사랑하는 4가지 기능

"이 제품이면 내 워크숍에 정말 도움이 되겠다"는 확신을 주는 기능들.

FEATURE 01

30초 합류, 4자리 코드

참여자 가입 안내에 5-10분 소비, 이제 끝.

운영자가 만든 4자리 코드와 닉네임만 입력하면 즉시 보드 진입. 가입도, 로그인도, 비밀번호도 없습니다. 학습자 누구도 도구 때문에 워크숍에서 빠지지 않습니다.

FEATURE 02

보드와 채팅, 한 화면에

분산되어 있던 흐름을 하나로.

좌측 보드 캔버스 + 우측 채팅 패널. 운영자가 카톡과 줌 채팅을 따로 켤 필요가 없습니다. 텍스트, 이미지, 파일, 링크 미리보기까지 자연스럽게 흐릅니다.

FEATURE 03

HRD 표준 템플릿 내장

KPT, 4F, 9 Window — 처음부터 한국 워크숍을 위해.

한국 HRD 현장에서 자주 쓰이는 회고·아이디어·액션러닝 템플릿을 기본 제공. 빈 보드부터 시작하지 않고, 검증된 구조 위에서 시작합니다.

FEATURE 04

Facilitator 모드

패들렛에는 없는, 운영자만의 권한.

타이머로 단계 진행, 채팅 공지 고정, 부적절한 메시지 모더레이션, 보드 잠금. 운영자가 워크숍 흐름을 완벽하게 통제할 수 있도록 설계된 권한 시스템.

자주 묻는 질문

결정 전 가장 많이 받는 5가지 질문에 답합니다.

네, 가능합니다. 운영자가 만든 4자리 보드 코드와 닉네임만 입력하면 즉시 보드에 진입합니다. 별도 계정 생성, 비밀번호, 이메일 인증이 필요 없습니다.
Fadlet은 보드 단위 데이터 격리, Firebase Security Rules 기반 권한 관리, 보드 코드 brute force 방어 등 기업 환경에 맞는 보안 체계를 갖추고 있습니다. 엔터프라이즈 플랜에서는 ISMS 인증, 온프레미스 옵션도 지원합니다.
네, BUSINESS 플랜 이상에서 세금계산서를 자동으로 발행해 드립니다. 토스페이먼츠를 통한 국내 카드·계좌이체 결제를 지원하며, 회계 부서에서 별도로 처리할 필요가 없습니다.
패들렛에서 Fadlet으로의 마이그레이션 도구를 준비 중입니다. 정식 출시 시점에 1-클릭으로 보드를 옮길 수 있는 기능을 제공할 예정입니다.
네, Fadlet은 처음부터 모바일 우선으로 설계되었습니다. PC에서는 보드+채팅 분할 화면, 모바일에서는 토글 방식으로 자연스럽게 전환됩니다. 워크숍 참여자의 60% 이상이 모바일로 접속한다는 점을 고려해 설계했습니다.
